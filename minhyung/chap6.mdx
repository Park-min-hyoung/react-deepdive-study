# 06장 리액트 개발 도구로 디버깅하기

## 리액트 개발 도구란?

- 리액트 팀은 리액트 애플리케이션의 원활한 개발을 위한 개발 도구인 react-dev-tools를 만들어 제공

## 리액트 개발 도구 설치

- 각 브라우저의 확장 도구를 통해 설치

## 리액트 개발 도구 활용하기

1. 컴포넌트

- 정적인 현재 리액트 컴포넌트 트리의 내용을 디버깅하기 위한 도구
- 컴포넌트 트리
- 컴포넌트명과 props
  - 컴포넌트명과 Key
  - 컴포넌트 도구
  - 컴포넌트 props
  - 컴포넌트 hooks
  - 컴포넌트를 렌더링한 주체, rendered by

2. 프로파일러

- 리액트가 렌더링하는 과정에서 발생하는 상황을 확인하기 위한 도구
- 설정 변경하기
- 프로파일링
  - 프로파일링 메뉴
  - Flamegraph
  - Ranked
- 타임라인
- 프로파일러로 렌더링 원인 파악해서 수정

## 정리

- 리액트 개발 도구를 활용하면 정적으로 생성된 컴포넌트 트리를 보는 것에서부터 프로파일링을 통해 리액트 애플리케이션이 시간이 지남에 따라 어떤 식으로 작동하는지, 불필요한 리렌더링이 일어나고 있는지 등을 확인할 수 있다.

# 발표

## 요소 탭

- 스타일 탭
  - 모든 스타일 정의되어있다.
  - 스타일 정의 패널 우측상단의 파일명 클릭하면 스타일 코드가 작성된 파일을 찾을 수 있다.(해당 코드 위치 확인)
- 계산됨 탭
  - 선택된 태그의 최종적으로 적용된 css 정보를 보여준다.
  - 계산됨 탭에서도 스타일의 위치를 파악할 수 있다.(속성의 dropdown 버튼 클릭하고 우측 파일명 클릭)
- 스타일 탭에서 필터 input 옆에 화살표가 있는 아이콘을 클릭하면 스타일 탭과 계산됨 탭을 같이 볼 수 있다.
- HTML 수정
  - 키보드로 DOM 트리 탐색(아래화살표: 아래로 이동, 위 화살표: 위로이동, 오른쪽 화살표: 아래 이동/노드 펼침, 왼쪽 화살표: 위쪽 이동/노드 닫음)
  - 특정 요소를 수정 했을 때 요소 우클릭 후 'HTML로 수정' 버튼 클릭 하면 수정한 HTML을 복사할 수 있다.
  - 특정 요소 선택 후 드래깅 함으로써 요소의 위치를 변경시킬 수 있다.
  - 특정 요소 선택 후 h 입력 시 요소를 숨길 수 있다.
- CSS 수정
  - 스타일 탭에 특정 선택자에 정의된 속성 앞 부분에 checkbox를 클릭해보며 적용 여부에 따라 스타일을 점검할 수 있다.
  - 속성값을 재정의 함으로써 여러 스타일의 적용해볼 수 있다.
  - 특정 선택자의 닫는 태그 라인을 클릭하면 새로운 속성을 추가할 수 있다.
  - 속성을 입력한 후 tab을 클릭하면 값을 입력할 수 있도록 커서가 이동된다.(속성 자동 완성 사용 시 자동으로 커서가 이동)
  - 코드에서 직접 인라인 스타일로도 적용할 수 있다.
  - 파일 경로를 클릭해서 수정한 CSS 코드를 복붙할 수 있다.
- 색상 선택 도구
  - 스타일 탭에서 색상 관련 속성은 색상 네모박스를 클릭하면 색상도구를 볼 수 있다.
  - 색상 도구의 스포이드 아이콘 클릭 시 현재 페에지에서 추출하고자 하는 색상을 추출할 수 있다.
- 가상 클래스 확인
  - 스타일에서 :hov 클릭하면 "요소 강제 상태 적용" 패널이 보인다.
  - 해당 패널에서 체크 박스 클릭을 통해 특정 가상 클래스가 적용된 선택자들을 볼 수 있다.
  - .cls 버튼을 클릭해서 특정 요소에 클래스를 추가할 수 있다.
- 박스 모델 확인하기
  - 계산됨 탭에서 요소의 크기, padding, border, margin의 크기를 확인할 수 있다.

## 명령어 입력(단축키 cmd + shift + p)

- 자바스크립트 비 활성화
  - "자바스크립트 사용 중지" 입력 후 클릭
  - 설정 탭의 디버거 항목에서도 가능
- 화면 캡처하기
  - "원본 크기 스크린샷" 입력 후 클릭(페이지 전체를 캡쳐할 수 있다)
  - body 태그에서 마우스 우클릭 후 노드 스크린샷 캡쳐 선택해도 동일함
  - 원하는 크기 캡쳐 가능

## 콘솔 탭

- 명령어

  - console.table(순회가능한 값)

    - 배열, 이중배열, 객체 등을 전달인자로 넘겨주면 테이블 형태로 데이터를 조회할 수 있다.
    - 객체의 경우 첫번째 전달인자에 객체 key 값 두번째 전달인자에 프로퍼티 키를 전달해 특정 프로퍼티 데이터를 조회할 수 있다.

    ```js
    const obj = { a: "1", b: "2" };
    const arr = [1, 2, 3, 4, 5];

    console.table(obj); // obj 조회
    console.table(arr); // arr 조회
    console.table([obj], ["a"]); // obj의 특정 property 조회
    ```

  - console.time(): 타이머 실행
  - console.timeEnd(): 타이머 중지, 소요시간 출력
  - $\_: 콘솔의 마지막 실행 결과 참조
    ```js
    "12345".split("").reverse(); // ['5', '4', '3', '2', '1']
    $_; // ['5', '4', '3', '2', '1']
    ```
  - 객체 복사
    - 콘솔에 보이는 객체에 마우스 우클릭 > object 복사 클릭
    - 콘솔창에 copy(객체 식별자명) 명령어 입력

- 영상 속도 조절하기
  - 아래 명령어를 콘솔에 입력시 영상이 n배의 속도로 재생된다.
  ```js
  funtion(){document.querySelector('video').playbackRate = n}()
  ```

## 로컬 스토리지

- 새로고침하거나 페이지를 종료하더라도 로컬 스토리지의 데이터는 영구적으로 유지된다.
- 관련 메소드(setItem, getItem, removeItem, clear: 모두 삭제), 프로퍼티(length: 키/값 쌍의 개수)
- 애플리케이션 탭에서 필터 기능을 통해 보다 빠르게 데이터를 찾을 수 있다.

## 네트워크 탭

- 모든 네트워크 활동을 기록
- "캐시 사용 중지" 기능을 사용해 더 정확하게 첫 방문자의 사용자 경험을 분석할 수 있다.
- 특정 열을 추가해서 추가적인 정보를 확인할 수 있다.(테이블 header에서 우클릭)
- 특정 네트워크를 클릭하면 상세 정보를 확인할 수 있다.
- 네트워크 속도를 컨트롤 할 수 있다.
- 매순간 렌더링 화면 확인하기
  - 아주 짧은 시간 단위로 페이지 렌더링 상태를 확인할 수 있다.(페이지 로딩 시간이 길 경우 문제를 해결해야 할 때 유용)
  - 우측 상단의 설정 버튼 클릭 > 스크린샷 체크박스 선택
